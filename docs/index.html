Multilingual CRUD fields

Use language dictionaries in the-api applications

`the-api-langs` is a module for APIs built with `the-api`. It contributes PostgreSQL migrations for `dict` and `languages`, then `the-api` uses `_lang` and `translate` to return localized fields from normal CRUD endpoints.

Before you start

Requirements

Node.js or Bun

Use Node.js 18+ or Bun 1+.

`the-api` installed

The host application runs on `the-api`.

`the-api-langs` installed

The module provides shared language migrations.

PostgreSQL tables

Create app tables and include `langs` so its migrations run.

Install

Package and environment

Install packages

bun add the-api the-api-langs

npm install the-api the-api-langs

Minimal `.env`

PORT=7788

DB_HOST=localhost
DB_PORT=5432

`langs` is migration-only. It does not expose HTTP routes by itself. Add CRUD routes for `dict` or `languages` when an app needs them.

Database

Tables used in examples

`the-api-langs` creates `dict` and `languages`. Your application owns tables such as `testNews`. 

CREATE TABLE IF NOT EXISTS "dict" (
  "id" SERIAL PRIMARY KEY,
  "textKey" INTEGER NOT NULL,
  "lang" VARCHAR(2) NOT NULL,
  "text" VARCHAR(4096) NOT NULL
);

CREATE UNIQUE INDEX IF NOT EXISTS dict_lang_text_key_idx
  ON "dict" ("lang", "textKey");

CREATE TABLE IF NOT EXISTS "languages" (
  "id" SERIAL PRIMARY KEY,
  "timeCreated" TIMESTAMPTZ NOT NULL DEFAULT now(),
  "timeUpdated" TIMESTAMPTZ NOT NULL DEFAULT now(),
  "code" VARCHAR(2) NOT NULL,
  "nativeName" VARCHAR(128) NOT NULL,
  "englishName" VARCHAR(128) NOT NULL,
  "textDirection" VARCHAR(3) NOT NULL DEFAULT 'ltr',
  "isActive" BOOLEAN NOT NULL DEFAULT true
);

CREATE UNIQUE INDEX IF NOT EXISTS languages_code_idx
  ON "languages" ("code");

CREATE TABLE IF NOT EXISTS "testNews" (
  "id" SERIAL PRIMARY KEY,
  "timeCreated" TIMESTAMP NOT NULL DEFAULT now(),
  "timeUpdated" TIMESTAMP NULL,
  "timeDeleted" TIMESTAMP NULL,
  "isDeleted" BOOLEAN NOT NULL DEFAULT false,
  "name" VARCHAR(4096) NOT NULL,
  "slug" VARCHAR(256) NULL,
  "body" TEXT NULL
);
TableOwnerPurpose
dictthe-api-langsStores source and translated text grouped by textKey.
languagesthe-api-langsStores language metadata such as names, direction, and active flag.
testNewsApplicationExample CRUD table with a translated name field.

Quickstart

Build a translated API

import { Routings, TheAPI, middlewares } from 'the-api';
import { langs } from 'the-api-langs';

const router = new Routings({ migrationDirs: ['./migrations'] });

router.crud({
  table: 'dict',
  prefix: 'langs',
});

router.crud({
  table: 'testNews',
  translate: ['name'],
  searchFields: ['name'],
});

const theAPI = new TheAPI({
  routings: [middlewares.common, langs, router],
});

await theAPI.up();

Request

curl 'http://localhost:7788/testNews?_fields=id,name&_lang=de'

Response

{
  "result": [
    { "id": 1, "name": "Hallo Welt" },
    { "id": 2, "name": "Oeffentliche API" }
  ],
  "error": false
}

How it works

Translation flow

Store canonical English text in the application table. Store every localized value in `dict` with the same `textKey`. 

INSERT INTO "dict" ("textKey", "lang", "text")
VALUES
  (1001, 'en', 'Hello world'),
  (1001, 'de', 'Hallo Welt'),
  (1001, 'es', 'Hola mundo')
ON CONFLICT ("lang", "textKey") DO UPDATE
SET "text" = EXCLUDED."text";

INSERT INTO "testNews" ("name", "slug")
VALUES ('Hello world', 'hello-world');
curl 'http://localhost:7788/testNews/1?_fields=id,name,slug&_lang=es'
{
  "result": {
    "id": 1,
    "name": "Hola mundo",
    "slug": "hello-world"
  },
  "error": false
}
curl 'http://localhost:7788/testNews?_fields=id,name&_lang=es&_search=Hola'
{
  "result": [
    { "id": 1, "name": "Hola mundo" }
  ],
  "error": false
}
The English row is the lookup bridge. `the-api` finds the `textKey` by matching the application field to the English `dict` row, then returns the requested language row.

Routes

CRUD endpoints you may expose

GET/testNews

List app rows with translated fields.

GET/testNews/:id

Read one app row with translated fields.

POST/langs

Create a dictionary record when `dict` uses `prefix: 'langs'`.

PATCH/langs/:id

Update one dictionary record.

DELETE/langs/:id

Delete one dictionary record.

GET/languages

List language metadata if exposed by the app.

Keep public reads open or protected according to the application. Production systems usually protect dictionary writes.

Examples

Common request and response pairs

Create a dictionary row

  -d '{"textKey":1003,"lang":"de","text":"Einstellungen"}'

Response

    "textKey": 1003,
    "lang": "de",
    "text": "Einstellungen"

curl 'http://localhost:7788/testNews?_fields=id,name&_lang=de&name=Hallo%20Welt'

Query language

`_lang` with normal the-api queries

ParameterMeaningExample
_langTranslate configured fields._lang=de
_fieldsSelect response fields._fields=id,name
_searchSearch configured `searchFields`._search=Hallo
_sortSort fields; `-field` means descending._sort=-id
_limit, _page, _skipPaginate result lists._limit=20&_page=2
name=valueFilter by source text or translated text, depending on `_lang`.name=Hallo%20Welt
Filtering follows the selected language. With `_lang=de`, `name=Hello%20world` does not match the translated `name`; use `name=Hallo%20Welt` or omit `_lang`.

Lifecycle

Migration behavior

`TheAPI` collects `migrationDirs` from every `Routings` instance in `routings`. Include `langs` and the language migrations are part of startup. 

const theAPI = new TheAPI({
  routings: [middlewares.common, langs, router],
});
MigrationPurpose
20240525132746_init_langs.jsCreates the old langs table.
20260427193323_add_languages.jsCreates languages.
20260427211407_rename_langs_to_dict.jsRenames langs to dict and renames indexes.

FAQ

Questions with request-answer examples

Does `langs` create routes?

import { langs } from 'the-api-langs';

const theAPI = new TheAPI({ routings: [langs] });
No. It registers migrations only. Add CRUD routes for
`dict` if the app needs `/langs` endpoints.

How do I read the original English text?

curl 'http://localhost:7788/testNews/1?_fields=id,name&_lang=en'
{
  "result": {
    "id": 1,
    "name": "Hello world"
  },
  "error": false
}

Can I use `en-US`?

INSERT INTO "dict" ("textKey", "lang", "text")
VALUES (1001, 'en-US', 'Hello world');
No with the current schema. `dict.lang` and
`languages.code` are VARCHAR(2). Use two-character codes
or add an application migration that changes the schema.

What happens if a translation is missing?

curl 'http://localhost:7788/testNews/1?_fields=id,name&_lang=fr'
{
  "result": {
    "id": 1,
    "name": "Hello world"
  },
  "error": false
}

Should I use `dict` or `langs`?

router.crud({ table: 'dict', prefix: 'langs' });
Use `dict` as the database table. Use `prefix: 'langs'`
only when you want the public HTTP path to be `/langs`.
